\hypertarget{md__2shr_2Github_2WheelTime_2notes_2hotkey-basics_autotoc_md4}{}\doxysubsection{\texorpdfstring{Params}{Params}}\label{md__2shr_2Github_2WheelTime_2notes_2hotkey-basics_autotoc_md4}

\begin{DoxyItemize}
\item {\ttfamily h\+Wnd}
\begin{DoxyItemize}
\item Window that receives {\ttfamily WM\+\_\+\+HOTKEY}
\item Can be {\ttfamily NULL} if using global hotkey, so that\textquotesingle{}s what we should do
\end{DoxyItemize}
\item {\ttfamily id}
\begin{DoxyItemize}
\item Unique identifier for the hotkey
\item Used to distinguish multiple hotkeys
\item We can just make this the index of the action on the wheel (so like 1, 2, 3, etc)
\end{DoxyItemize}
\item {\ttfamily fs\+Modifiers}
\begin{DoxyItemize}
\item Modifier keys bitmask:\+
\begin{DoxyItemize}
\item {\ttfamily MOD\+\_\+\+ALT} (alt key)
\item {\ttfamily MOD\+\_\+\+CONTROL} (ctrl key)
\item {\ttfamily MOD\+\_\+\+SHIFT} (shift key)
\item {\ttfamily MOD\+\_\+\+WIN} (win key)
\end{DoxyItemize}
\item Using any of these means this key would also have to be pressed for the hotkey to work (kinda limited?)
\end{DoxyItemize}
\item {\ttfamily vk}
\begin{DoxyItemize}
\item Virtual key code (e.\+g. {\ttfamily VK\+\_\+\+SPACE}, {\ttfamily VK\+\_\+O}, etc)
\item We would have to collect the keypresses from the user and then convert them to virtual key codes
\end{DoxyItemize}

To keep the main thread free, we will probably have to put this loop on a separate thread so it keeps everything else responsive.\hypertarget{md__2shr_2Github_2WheelTime_2notes_2hotkey-basics_autotoc_md9}{}\doxysection{\texorpdfstring{Error Handling}{Error Handling}}\label{md__2shr_2Github_2WheelTime_2notes_2hotkey-basics_autotoc_md9}

\begin{DoxyItemize}
\item Hotkey fails if already registered by another app
\item Must check return value of {\ttfamily Register\+Hot\+Key} since it returns a boolean
\item Some system combos are reserved (Win+L, Ctrl+\+Alt+\+Del)
\item {\ttfamily Get\+Last\+Error()} will return the error code if the return is false, most common error for us probably will be {\ttfamily ERROR\+\_\+\+HOTKEY\+\_\+\+ALREADY\+\_\+\+REGISTERED}
\item We should probably have a way to gracefully handle these errors 
\end{DoxyItemize}
