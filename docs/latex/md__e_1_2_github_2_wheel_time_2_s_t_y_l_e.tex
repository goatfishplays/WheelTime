Conventions for active code under {\ttfamily app/\+}, {\ttfamily platform/\+}, and {\ttfamily launcher/\+}. When older code disagrees with this file, new and touched code should move toward these rules. Treat {\ttfamily \doxylink{_scheduler_8hpp}{Scheduler.\+hpp}} /\+ newer Action\+Items as the reference shape.

\hypertarget{md__e_1_2_github_2_wheel_time_2_s_t_y_l_e_autotoc_md176}{}\doxysection{\texorpdfstring{Files and directories}{Files and directories}}\label{md__e_1_2_github_2_wheel_time_2_s_t_y_l_e_autotoc_md176}

\begin{DoxyItemize}
\item Headers:\+ {\ttfamily .hpp} + {\ttfamily \#pragma once} (no include guards).
\item Sources:\+ {\ttfamily .cpp}.
\item Names:\+ {\bfseries{Pascal\+Case}} matching the primary type ({\ttfamily \doxylink{_scheduler_8hpp}{Scheduler.\+hpp}}, {\ttfamily \doxylink{_mouse_button_8cpp}{Mouse\+Button.\+cpp}}).
\item Domains as folders:\+ {\ttfamily Action\+Items/\+}, not {\ttfamily action\+\_\+items/\+}.
\item Tests:\+ {\ttfamily app/\+tests/\+\texorpdfstring{$<$}{<}topic\texorpdfstring{$>$}{>}\+\_\+\texorpdfstring{$<$}{<}kind\texorpdfstring{$>$}{>}.cpp} with CMake target of the same stem.
\begin{DoxyItemize}
\item Use {\ttfamily \texorpdfstring{$\ast$}{*}\+\_\+tests} for kept regression suites; {\ttfamily \texorpdfstring{$\ast$}{*}\+\_\+smoke} only for disposable phase checks.
\item Prefer {\ttfamily action\+\_\+items} over glued {\ttfamily actionitems} in new names.
\end{DoxyItemize}
\item Assets /\+ JSON:\+ {\bfseries{snake\+\_\+case}} or existing resource names; do not force Pascal\+Case onto wire formats.
\end{DoxyItemize}

\begin{DoxyItemize}
\item Classes, structs, enums, type aliases:\+ {\bfseries{Pascal\+Case}}.
\item Prefer {\ttfamily enum class} with Pascal\+Case enumerators.
\item Action\+Item implementations:\+ prefix {\bfseries{{\ttfamily AI\+\_\+}}} + Pascal\+Case remainder.
\begin{DoxyItemize}
\item Good:\+ {\ttfamily AI\+\_\+\+Keystroke}, {\ttfamily AI\+\_\+\+Mouse\+Button}, {\ttfamily AI\+\_\+\+Nth\+Recent}.
\item Bad:\+ {\ttfamily AI\+\_\+nth\+Recent} (lowercase after prefix).
\end{DoxyItemize}
\item Avoid unexplained abbreviations in new public API ({\ttfamily Input\+Receiver} over {\ttfamily Input\+Rcvr} for new names; existing abbreviations may stay until renamed intentionally).
\end{DoxyItemize}\hypertarget{md__e_1_2_github_2_wheel_time_2_s_t_y_l_e_autotoc_md180}{}\doxysubsection{\texorpdfstring{Functions and methods}{Functions and methods}}\label{md__e_1_2_github_2_wheel_time_2_s_t_y_l_e_autotoc_md180}

\begin{DoxyItemize}
\item {\bfseries{camel\+Case}}.
\item Booleans:\+ prefer {\ttfamily is} /\+ {\ttfamily has} /\+ {\ttfamily can} prefixes ({\ttfamily is\+Paused}, {\ttfamily is\+Cancelable}).
\item Accessors:\+ pick {\bfseries{one}} style per type and stick to it.
\begin{DoxyItemize}
\item Preferred for library types:\+ bare nouns for trivial getters ({\ttfamily channel()}, {\ttfamily name()}), or consistent {\ttfamily get\+X()} — not both for the same field.
\item Do not add a second alias ({\ttfamily get\+Items()} {\bfseries{and}} {\ttfamily items()}).
\end{DoxyItemize}
\item Prefer full words for mutators/\+size:\+ {\ttfamily remove\+Action}, {\ttfamily item\+Count} /\+ {\ttfamily num\+Items} — avoid {\ttfamily rem\+Action}, {\ttfamily len} in new or refurbished API.
\item Parameters:\+ no leading underscore ({\ttfamily input\+Bind}, not {\ttfamily \+\_\+input\+Bind}).
\end{DoxyItemize}\hypertarget{md__e_1_2_github_2_wheel_time_2_s_t_y_l_e_autotoc_md181}{}\doxysubsection{\texorpdfstring{Members and locals}{Members and locals}}\label{md__e_1_2_github_2_wheel_time_2_s_t_y_l_e_autotoc_md181}

\begin{DoxyItemize}
\item Private /\+ protected data members:\+ {\bfseries{{\ttfamily m\+\_\+}}} + camel\+Case ({\ttfamily m\+\_\+mailbox}, {\ttfamily m\+\_\+channel}).
\item Public fields are allowed only for small DTOs /\+ Action\+Item value fields ({\ttfamily AI\+\_\+\+Delay::\+duration}). Document that choice on the type; do not mix {\ttfamily m\+\_\+} and bare private fields on the same class.
\item Locals and parameters:\+ camel\+Case, no {\ttfamily m\+\_\+}.
\item Constants:\+ {\ttfamily k\+Camel\+Case} or {\ttfamily Pascal\+Case} enum /\+ {\ttfamily constexpr}; avoid {\ttfamily SCREAMING\+\_\+\+SNAKE} except for macros.
\end{DoxyItemize}\hypertarget{md__e_1_2_github_2_wheel_time_2_s_t_y_l_e_autotoc_md182}{}\doxysubsection{\texorpdfstring{Domain vocabulary}{Domain vocabulary}}\label{md__e_1_2_github_2_wheel_time_2_s_t_y_l_e_autotoc_md182}

\begin{DoxyItemize}
\item One name per concept across app and platform boundaries.
\begin{DoxyItemize}
\item Example debt:\+ cancel “latest” vs “most recent” — consolidate when those headers are edited.
\end{DoxyItemize}
\item JSON /\+ config type tags:\+ {\bfseries{snake\+\_\+case}} ({\ttfamily launch\+\_\+app}, {\ttfamily nth\+\_\+recent}, {\ttfamily hotkey}).
\begin{DoxyItemize}
\item Prefer a single canonical tag per item kind; do not add duplicate aliases ({\ttfamily hotkey} vs {\ttfamily keystroke}) without a migration plan.
\end{DoxyItemize}
\end{DoxyItemize}

\begin{DoxyItemize}
\item Prefer clear ownership:\+ {\ttfamily unique\+\_\+ptr} for polymorphic sequences; menus reference actions by stable id.
\item Mark side-\/effect-\/free queries {\ttfamily \+[\+[nodiscard]\+]\+} when ignoring the result is a bug ({\ttfamily cancelable()}, {\ttfamily execute} results).
\item Prefer {\ttfamily noexcept} on trivial getters/\+setters and moves when accurate.
\item Out parameters and boolean “success” returns:\+ prefer typed results ({\ttfamily Execute\+Result}) over ambiguous {\ttfamily bool} when already used nearby.
\item Units in names or comments when non-\/obvious ({\ttfamily duration\+Ms} vs bare {\ttfamily duration}); do not silently mix seconds and milliseconds for similar fields.
\end{DoxyItemize}

\hypertarget{md__e_1_2_github_2_wheel_time_2_s_t_y_l_e_autotoc_md190}{}\doxysection{\texorpdfstring{Formatting and language}{Formatting and language}}\label{md__e_1_2_github_2_wheel_time_2_s_t_y_l_e_autotoc_md190}

\begin{DoxyItemize}
\item Language:\+ C++23 for active CMake targets unless a subdirectory documents otherwise. Do not quietly lower the standard on new targets.
\item Indentation:\+ 4 spaces (match surrounding file).
\item Braces:\+ Allman-\/style /\+ existing file style — {\bfseries{match the file you edit}}; do not reformat untouched regions for preference alone.
\item Namespace body:\+ prefer types at column 0 inside {\ttfamily namespace \doxylink{namespace_application}{Application} \{ … \}} (scheduler style). When editing heavily indented older GUI headers, migrate the touched type toward that layout only if the diff stays focused.
\item Prefer {\ttfamily nullptr}, {\ttfamily override}, {\ttfamily = default} /\+ {\ttfamily = delete} where appropriate.
\end{DoxyItemize}

A {\ttfamily .clang-\/format} may be added later to encode brace/\+indent rules; until then, match neighboring code.

\item Abbreviated \doxylink{namespace_platform}{Platform} names ({\ttfamily Input\+Rcvr}) retained for ABI chill — rename when convenient
\item Generated Doxygen under {\ttfamily docs/\+html} may still mention obsolete names until regenerated
\end{DoxyEnumerate}
