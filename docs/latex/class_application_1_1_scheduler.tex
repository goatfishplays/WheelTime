\doxysubsection{Detailed Description}
Accepts Actions and runs them via \doxylink{class_application_1_1_worker_pool}{Worker\+Pool} with channel + delay policy. 

The scheduler is the only component that makes scheduling decisions. Callers submit work from any thread; a dedicated scheduler thread owns every \doxylink{class_application_1_1_action_execution_context}{Action\+Execution\+Context}, enforces channel FIFO via \doxylink{class_application_1_1_channel_manager}{Channel\+Manager}, parks Delayed contexts on \doxylink{class_application_1_1_delay_queue}{Delay\+Queue}, and dispatches runnable work to \doxylink{class_application_1_1_worker_pool}{Worker\+Pool}.

Channel rules:\+
\begin{DoxyItemize}
\item At most one \doxylink{class_application_1_1_action}{Action} in-\/flight per channel key.
\item Delayed Actions {\bfseries{hold}} their channel until wake (strict sequentialness).
\item Waiting Actions preserve earliest\+Start (nested future schedules stay correct).
\item Channel 0 assigns a private key per \doxylink{class_application_1_1_action}{Action} so those Actions never block one another.
\end{DoxyItemize}

Delay rules:\+
\begin{DoxyItemize}
\item Workers never sleep; Delay\+Until returns the context to the scheduler.
\item The scheduler thread waits on its mailbox until a new event, the earliest \doxylink{class_application_1_1_delay_queue}{Delay\+Queue} wake time, or stop.
\item While paused, due delays are not dispatched; on resume, pending wake times are shifted forward by the pause duration so remaining delay is preserved.
\end{DoxyItemize}

Cancellation:\+
\begin{DoxyItemize}
\item cancel\+Action /\+ cancel\+Channel /\+ cancel\+All (thread-\/safe).
\item Only between Action\+Items; the current item always finishes.
\item Actions with cancelable()==false are skipped.
\item set\+Cancel\+Flush Actions run immediately on cancel (e.\+g. Key\+Up).
\item schedule\+Action(..., remove\+If\+Parent\+Cancelled=true) links delayed cleanup that is discarded when the parent is cancelled (after flush).
\item Cancel-\/flush bypasses pause so hardware cleanup can still run.
\end{DoxyItemize}

Pause/\+resume:\+
\begin{DoxyItemize}
\item \doxylink{class_application_1_1_scheduler_a80cbf8adf69eb5df907c5a96f1361bda}{pause()} stops dispatch of new work; submits are still admitted.
\item Delayed Actions stop progressing (virtual clock).
\item Workers finish the current \doxylink{class_application_1_1_action_item}{Action\+Item} then sleep until resume. 
\end{DoxyItemize}
